Ce fichier a pour but de vous presenter rapidement les possibilités d'Octave 
version 2 et MatLab 5, alternatives a APL.

Octave/Matlab est un langage mathématique interactif de haut niveau pour le 
calcul numérique. Il utilise une ligne de commande et réalise les calculs au 
fur et à mesure que vous les demandez, ou à partir d'un programme écrit.
Octave/Matlab est particulierement adapté à la resolution de
problèmes numériques non linéaires et d'algèbre linéaire.

Note : Octave est compatible MatLab(R), produit commercial très répandu, et le
langage Octave/MatLab est directement inspiré du langage C. Ne paniquez pas,
c'est très simple !
Notons que certaines syntaxes (",**,++,--,!,**,#,&&,||) sont propres à Octave, mais on 
est pas obligé de les utiliser.

Pour toute remarque sur ce travail, cliquez ICI.

Démarrer avec Octave.

  • Pour appeler Octave, il suffit de taper la commande "octave" sur votre terminal a partir d'un IBM : jullien, tran, anglaret, foret, vacher.
  • Pour appeler Matlab, tapez "matlab" à partir de athena .
Il apparait alors un 'prompt' du type "octave:1>" ou ">>" indiquant que le programme attend vos instructions. Les élements qu'Octave/Matlab sait manipuler sont des variables (entières, réelles, complexes, matrices, vecteurs, chaines de caractères) et des fonctions. Les resultats sont affichés par le programme 'GnuPlot' sous Octave, ou directement par Matlab. IMPORTANT : Dans tous les cas, si une commande d'Octave/Matlab vous est obscure, il suffit de taper 'help ' pour que s'affiche une description rapide de ce qu'elle fait. Pour obtenir la liste de tout ce que connait Octave/Matlab, tapez "help" au prompt. Il existe des aides 'interactives' par netscape avec les commandes suivantes : doc : aide Octave et Matlab generale. ldvintro : cette introduction ldv : les commandes de traitement du signal 'maison'. IMPORTANT 2 Le retour : Ce guide n'est qu'indicatif. Pour de plus amples informations, consultez le mode d'emploi complet, ou venez me voir... Nous allons maintenant nous familliariser avec les commandes Octave/Matlab en suivant une serie d'exemples.

Les types de données.

Interessons nous à Ignace (ou Germaine pour les feministes). Il vient de récuperer les résultats du futur prix Nobel et veut les exploiter. Il communique ses precieux resultats à octave/matlab : (les lignes de commande sont representées par ':n> commande') commande commentaire :1> age_du_capitaine = 35.25 un réel, qui est affiché à l'ecran. :2> shtroumpf = 1 + 2i un complexe. (NB : 4i = 4*i) :3> loto = [ 2 78 6 95 15 ] un vecteur ligne. :4> zorglub = 5:15 un intervalle de 5 à 15 par pas de 1. :5> tobozo = [ 1 i ; 3 4*i ] une matrice. Le ';' indique un saut de ligne. Les lignes de commande entrées sont directement interpretées, et le resultat est mis dans la variable 'ans' (qui contient donc toujours le dernier resultat). Si la ligne de commande ne se termine pas par un point virgule (;), le resultat est affiché. La variable 'A' est différente de la variable 'a'. Note pour Octave : L'appui sur la touche 'TAB' lors de la frappe de la commande demande à Octave de completer la ligne d'apres les mots et fonctions qu'il connait. Un double appui sur cette touche liste les commandes commencant par les lettres que vous avez tapées. Exemple : tapez sur Octave (le pauvre) : 'polyd' puis sur 'TAB'. Octave complète en ajoutant 'eriv'. Essayez avec 'po' puis double-'TAB' -> octave liste les commandes commencant par 'po'. La ligne de commande Octave est éditable, éffacable, et est enregistrée dans un fichier '.octave-hist' sur votre racine. L'ensemble des sessions octave/matlab peuvent etre intégralement enregistrées dans le fichier 'diary' en tappant 'diary on' (ou 'off' pour desactiver).

Les opérateurs.

Les opérateurs mathématiques de base sont +, -, * (multiplication), / (division). L'élévation à une puissance s'écrit '^' (et aussi '**' avec Octave). Lorsque l'on fait preceder un opérateur par un point (.), on indique à oct/mat que l'opération doit etre effectuée terme a terme. Cela est bien utile pour effectuer par exemple des calculs rapides sur des vecteurs. \ (anti-slash) : division à gauche (comme / mais sans calculer l'inverse pour les matrices). ' (quote) : transposition de vecteur ou matrice. ++ : incrémentation (x=x+1) (Octave) -- : décrémentation (x=x-1) (Octave) - : négation 'normale' = : égalite 'normale' (assignation) [ x y ... ] : concaténation des élements 'x' 'y' ... ; : passage à la ligne dans une concaténation avec [ ]. 'toto' : chaine de charactères (et aussi "toto" avec Octave) : : intervalle ou colonne/ligne. ex 1:10 = [ 1 2 3 .. 10 ] , ou 1:0.5:10 = [ 1 1.5 2 .. 10 ] % : commentaires (et aussi '#' avec Octave) . : opération terme à terme Les opérateurs de comparaison et booléens sont les suivants : <, >, <=, >= : devinez == : égalite logique (test) ~=, <> : different (et != avec Octave) & : et | : ou ~ : négation logique ('non ...') (et aussi '!' avec Octave) Certains opérateurs sont un peu spéciaux. Prenons par exemple le ':'. Il indique de facon générale un intervalle. ':' : de 1 à la taille de l'objet qu'il indexe. (ligne ou colonne) 'n:m' : de n à m par pas de 1. 'n:p:m' : de n à m par pas de p. Les opérateurs 'increments' réalisent une variation de 1 sur une variable. Placés avant la variable, ils incrementent avant l'évaluation (Octave only). Les lignes sont évaluées de gauche à droite, en tenant compte de l'ordre de priorité habituel.

Voir vos donnees.

Ignace desire presenter ses resultats. Tout d'abord il regarde quelles sont les variables qu'il a communiqué à Octave/Matlab, avec la comande 'who'. Puis pour voir une variable, il tape son nom : octave:8> loto affiche 'loto = 2 78 6 95 15' si l'on desire voir une partie d'une variable, il suffit de le preciser : octave:9> tobozo(:,2) prend la deuxieme colonne de 'tobozo' et affiche ans = 0 + 1i 0 + 4i Lors d'un affichage sur plusieures pages, appuyez sur 'Return' et 'space' pour faire defiler les lignes, sur 'q' pour arreter l'affichage. Crtl+C pour stopper. Pour créer un graphique, il suffit de taper 'plot(y)' (x est alors 1:n) ou 'plot(x,y)'. On peut aussi utiliser 'disp(loto)' ou fprintf(stdout,"%f",loto) (comme en C).

Appeler des fonctions et faire des calculs.

Octave/Matlab ne serait pas grand chose sans ses bibliothèques. Elles comprennent de nombreuses fonctions specialisées de calcul. Certaines font partie intégrante d'Octave/Matlab (situées au niveau de l'orteil du pied gauche), et d'autres sont des fichiers de fonctions éditables. Les fichiers de fonction sont des fichiers textes, d'extension '.m', dont le nom est le nom de la fonction ecrite dedans. Ils sont constitués de suites d'instructions comme si elles étaient tapées directement à la ligne de commande Prenons un exemple : Tapez sur un terminal IBM (pas octave) 'faxe /usr/local/GNU/Octave/lib/octave/1.1.0/m/signal/sinc.m' sur IBM ou 'faxe /usr/local/MatLab/toolbox/local/FromOctave/sinc.m' sur Athena et voici que par magie, vous editez la fonction 'sinc'. Cela est aussi possible en tapant 'type sinc' avec Oct/Mat. On voit qu'elle demande en entrée une valeur (x) et retourne un resultat (result). Puis se trouvent des lignes de commentaires, ceux qui sont affichés lorsque l'on tape 'help sinc'. La taille de l'élément d'entrée est calculée (en lignes et colonnes), puis le nombre d'éléments. La variable 'result' est initialisée à 1 (il n'est pas toujours necessaire d'initialiser les variables, car elles sont crées dès que le besoin s'en fait resentir. Elle sont aussi de taille variable). La fonction 'find' est appelée, et retourne la position des élements non nuls. Pour tous ces élements, le sinus cardinal est calculé, puis le resultat est mis sous la forme du vecteur d'entrée. Pour faire vos propres calculs, vous utiliserez des fonctions, et vous en ecrirez vous meme. Pour appeler un fichier '.m' il suffit de taper son nom. Ex : sinc(loto) Nous detaillerons plus loin les bibliothèques disponibles.

Programmation structuree.

Les programmes 'octave/matlab' peuvent (et doivent) etre structurés par des instructions de controle. Voici la liste des mots clef correspondants, et leur description : if (condition) % TEST (then-body) % execute (body) si (condition) est vraie (i.e different de 0) end on peut aussi utiliser : if (condition) (then1) elseif (condition2) % équivalent à 'else' puis 'if' (then2) else (else-body) % execute si les tests sont tous faux end while (condition) % TANT QUE (body) % execute (body) tant que (condition) est vraie end for var = expression(vecteur ou matrice) % BOUCLE (for-body) % à chaque pas, var = une colonne de 'expression' end Il est possible de sortir des structures 'while' et 'for' avant terme (i.e au milieu de l'execution) par l'instruction 'break'. L'instruction 'continue' permet de passer directement au pas suivant (Octave only). La déclaration des fonctions se fait de la facon suivante : function retval = function_name(arguments) % ou [ retval1, retval2 .. ] = function_name(arguments) (function body) end Le nombre d'arguments et de valeurs à retourner en fin de fonction peut etre variable. Ces nombres sont stockes dans 'nargin' et 'nargout'. Cela permet par exemple de faire des fonctions qui se comportent differemment suivant le nombre d'arguments passés, ou attendus en fin d'execution. Un fichier texte commencant par 'function' est par defaut le fichier definissant la fonction dont le nom est celui du fichier. Tout autre fichier texte est interpreté comme une suite d'instructions octave/matlab comme si elles étaient tapées à la main.. Pour le lancer il suffit de taper le nom du fichier sans l'extension '.m'.

Vos fichiers de données

Octave et Matlab utilisent un format '.mat' de stockage binaire. Une fois les données en mémoire, on tape : save -mat-binary filename.mat [variables] avec Octave save filename [variables] avec Matlab pour sauver sur disque.

 Pour charger un fichier '.mat' en mémoire, on tape : load filename.mat [variables] avec Octave load filename [variables] avec Matlab En ce qui concerne l'importation de données exterieures (expèriences par exemple), je vous conseille de lire Traitement de donnees avec Octave/Matlab(voir 'Importation de données').
